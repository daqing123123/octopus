# 🐙 八爪鱼 Octopus

> **个人AI触手 × 企业协作平台**
> 每个员工都有独立的AI助手（触手），企业是八爪鱼本体，触手连接本体，协同进化。

---

## 💡 核心理念

### 为什么叫"八爪鱼"？

传统企业软件是"人适应系统"，员工被迫学习各种复杂工具。

**八爪鱼反过来了：**

```
┌─────────────────────────────────────────────────────────┐
│                    🐙 八爪鱼（企业）                     │
│                                                         │
│    ┤ 触手1 ── 触手2 ── 触手3 ── 触手4 ── 触手5 ┤        │
│      │        │        │        │        │            │
│      ↓        ↓        ↓        ↓        ↓            │
│    员工A    员工B    员工C    员工D    员工E           │
│                                                         │
│    每个触手 = 员工专属AI助手                            │
│    八爪鱼 = 企业协作大脑                                │
└─────────────────────────────────────────────────────────┘
```

### 触手是什么？

**触手（Claw）= 员工的私人AI助手**

- 🧠 **跟随人走**：你换了公司，触手跟你走，记忆、习惯、技能全保留
- 🔒 **隐私可控**：你决定哪些数据共享给企业，哪些只属于自己
- 🤖 **主动服务**：触手会主动提醒、建议、执行，而不是等你说
- 📈 **越用越懂你**：学习你的工作习惯，变成"第二个你"

### 八爪鱼是什么？

**八爪鱼（Octopus）= 企业协作平台**

- 🏢 **企业本体**：连接所有触手，形成协作网络
- 🧩 **功能组件**：日程、文档、表格、任务、OKR、知识库、审批、会议...
- 🧠 **企业大脑**：聚合所有触手的智慧，形成企业级AI能力
- 📊 **习惯池**：匿名汇总员工习惯，发现最佳实践

### 触手 ↔ 八爪鱼 关系

```
员工入职前                    员工在职期间                  员工离职后
    │                             │                          │
    ▼                             ▼                          ▼
┌────────┐                  ┌────────────┐              ┌────────┐
│ 个人触手 │ ──加入企业──→ │ 触手+企业连接 │ ──离开企业──→ │ 个人触手 │
│        │                  │            │              │        │
│ 完全私有│                  │ 部分数据共享│              │ 完全私有 │
└────────┘                  └────────────┘              └────────┘
                                │
                                ▼
                         企业获得：
                         - 员工公开的数据
                         - 匿名习惯数据
                         - AI协作能力
```

**关键点：**
- 触手永远属于员工，不属于企业
- 员工决定共享什么，企业不能强制
- 离职 = 断开连接，触手带走，企业数据留下

## 🚀 快速开始

### 环境要求

- Node.js 18+
- PostgreSQL 14+（或使用云数据库）
- Redis 6+（或使用云Redis）

### 安装步骤

```bash
# 1. Clone项目
git clone https://github.com/你的用户名/octopus.git
cd octopus

# 2. 安装依赖
cd src/backend && npm install
cd ../frontend && npm install

# 3. 配置环境
cp .env.example .env
# 编辑 .env 填入你的数据库配置

# 4. 运行数据库迁移
cd src/backend
npm run migrate

# 5. 启动服务
npm run dev  # 后端
# 新开终端
cd src/frontend && npm run dev  # 前端
```

### 访问地址

- 前端：http://localhost:3001
- 后端API：http://localhost:3000
- API文档：http://localhost:3000/docs
